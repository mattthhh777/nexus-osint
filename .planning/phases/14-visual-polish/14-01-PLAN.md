## Section 1 — Design System Audit

### `tokens.css` (127 linhas)
**O que faz**: fonte única da verdade. Define `--color-accent` (#f0a030), severity 4-tier (critical/high/medium/low), semantic (success/info), text 4-tier, typography (Space Grotesk / JetBrains Mono / Inter), scale 8px, radii tight, shadows, durations, z-index.
**Tokens existentes relevantes**: todos os `--color-*-muted` e `--color-*-border` já existem para cada severidade. `--shadow-glow` usa accent. `--color-info` (#3b82f6) disponível mas não ideal para "error lookup failed" (muito azul, lê como link).
**O que precisa mudar**:
- Adicionar `--color-error` (distinto de `--color-critical` — semanticamente diferente: critical = "dado grave", error = "falha de I/O"). Sugestão: `#64748b` (slate-500) ou `#94a3b8` (slate-400) para acumular "muted/failed" signal, não "alarm".
- Adicionar `--color-error-muted` e `--color-error-border` para consistência com outras famílias.
- Adicionar `--radius-xl: 10px` (cantos maiores para dropdown de user menu, que precisa se destacar).
- Opcional: `--shadow-dropdown: 0 12px 32px rgba(0,0,0,.55), 0 0 0 1px var(--color-border-strong) inset;` para menu dropdown.

**Implementação**:
```js
function formatTimestamp(iso) {
  if (!iso || typeof iso !== 'string') return '─';
  const d = new Date(iso.length === 16 ? iso + ':00Z' : iso);
  if (isNaN(d.getTime())) return '─';
  const fmt = new Intl.DateTimeFormat('en-US', {
    month: 'short', day: 'numeric', year: 'numeric',
    hour: '2-digit', minute: '2-digit', hour12: false,
    timeZone: 'UTC',
  });
  const parts = fmt.formatToParts(d);
  const get = t => parts.find(p => p.type === t)?.value || '';
  return `${get('month')} ${get('day')}, ${get('year')} · ${get('hour')}:${get('minute')} UTC`;
}
```
**Teste manual**: `formatTimestamp('2026-04-18T04:40:00Z')` = `"Apr 18, 2026 · 04:40 UTC"`.
**Teste fallback**: `formatTimestamp('2026-04-18T04:40')` (truncado, sem Z) = `"Apr 18, 2026 · 04:40 UTC"` (assume UTC via append).
