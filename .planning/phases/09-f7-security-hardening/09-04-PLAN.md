<task type="auto">
  <name>Task 2: Apply strict CSP + D-13 security headers in nginx.conf</name>
  <files>nginx.conf</files>
  <read_first>
    - nginx.conf (current full content — locate server block, /css/, /js/ location blocks; identify existing add_header directives)
    - .planning/phases/09-f7-security-hardening/09-CONTEXT.md §D-02, D-03, D-13
    - planning_guidance section in this prompt (nginx add_header inheritance trap)
  </read_first>
  <action>
    1. **Inheritance trap fix.** nginx `add_header` directives are NOT inherited by a location block if
       that location block defines its OWN `add_header`. Currently `/css/` and `/js/` location blocks
       define add_header (e.g., Cache-Control) which shadows the server-block headers.

       Fix: in EACH `/css/`, `/js/` location block, REPEAT the full security header set
       (or use an `include` snippet — preferred).

       Create the repeated set as a snippet block (or inline if include files aren't used in this project).

    2. **Server block headers** — add to the main `server { ... }`:
       ```nginx
       add_header Content-Security-Policy "default-src 'self'; script-src 'self'; style-src 'self'; img-src 'self' data:; connect-src 'self'; font-src 'self'; object-src 'none'; base-uri 'self'; form-action 'self'; frame-ancestors 'none'" always;
       add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;
       add_header X-Content-Type-Options "nosniff" always;
       add_header X-Frame-Options "DENY" always;
       add_header Referrer-Policy "strict-origin-when-cross-origin" always;
       add_header Permissions-Policy "geolocation=(), microphone=(), camera=()" always;
       ```
       The `always` flag ensures headers are sent on error responses too (4xx/5xx).
       Note: NO `unsafe-inline` in script-src or style-src. NO HSTS preload (D-13 explicit).

    3. **Repeat the same six add_header lines verbatim inside `/css/` and `/js/` location blocks.**
       This fixes the inheritance trap. If those location blocks don't currently exist as separate
       blocks, add explicit `location /css/ { ... }` and `location /js/ { ... }` blocks containing
       the full header set + the necessary try_files / root directives.

    4. Preserve existing nginx config: limit_req zones (D-08 outer DDoS shield), real_ip_module config,
       proxy_pass directives — DO NOT touch any of these. Only add headers.

    5. Validate config syntax before reloading:
       ```bash
       docker compose exec nexus nginx -t
       # OR if nginx runs on host: sudo nginx -t
       ```
  </action>
  <verify>
    <automated>nginx -t -c $(pwd)/nginx.conf 2>&1 | grep -q "syntax is ok"</automated>
  </verify>
