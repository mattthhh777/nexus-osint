    1. Pick naming convention: **kebab-case** for data-action values (e.g., `data-action="delete-case"`, `data-action="export-csv"`). Document at top of state.js with a comment block listing every action name registered.

    2. In static/js/state.js, add (or extend) a single delegated dispatcher:
       ```javascript
       // Action registry: data-action="kebab-name" → handler(target, dataset)
       const ACTIONS = Object.create(null);
       export function registerAction(name, handler) { ACTIONS[name] = handler; }

       function handleAction(ev) {
         const el = ev.target.closest('[data-action]');
         if (!el) return;
         const name = el.dataset.action;
         const fn = ACTIONS[name];
         if (!fn) { console.warn('unregistered action:', name); return; }
         ev.preventDefault();
         fn(el, el.dataset, ev);
       }

       export function initDelegation() {
         document.addEventListener('click', handleAction);
       }
       ```
       Export both `registerAction` and `initDelegation`. Other modules will call `registerAction('delete-case', deleteCase)` at module load.

    3. Create static/js/bootstrap.js. Move the contents of the inline `<script>init();</script>` block from static/index.html lines 350-359 (per RESEARCH.md range — verify exact lines on read) into bootstrap.js. It should:
       ```javascript
       import { initDelegation } from './state.js';
       // ...whatever the inline init() did...
       document.addEventListener('DOMContentLoaded', () => {
         initDelegation();
         init();   // existing init function
       });
       ```
       Adjust import paths as needed (the project uses module scripts or plain scripts — match existing convention).

    4. In static/index.html: REMOVE the inline `<script>init();</script>` block entirely. Add at the end of the body (before `</body>`):
       ```html
       <script type="module" src="/js/bootstrap.js"></script>
       ```
       (or non-module if other scripts are non-module — match the existing pattern in the file).

    5. In static/admin.html: if any inline `<script>` blocks contain executable code, extract them to a sibling `static/js/admin-bootstrap.js` and reference via external `<script src=>`. Inline `<script src="...">` is fine (CSP only blocks inline content, not external src).
