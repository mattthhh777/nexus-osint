# Plan 06-01: Memory Guards + Sherlock Async + Health Instrumentation

## Goal

Enforce memory discipline across the NexusOSINT backend: cap unbounded serialization, bound Sherlock HTTP responses, convert Sherlock to native async (eliminate deprecated event loop), and add tracemalloc instrumentation with enriched `/health` and new `/health/memory` endpoints.

## Pre-conditions

- Phase 11 complete: OathnetClient singleton, TTLCache, read_stream wired, httpx-only
- Phase 05 complete: TaskOrchestrator ready (not wired to _stream_search — out of scope)
- VPS constraint: 1vCPU / 1GB RAM / 800MB Docker limit

## Tasks

### T1 — Breach serializer memory guard (api/main.py)

Add `MAX_BREACH_SERIALIZE = 200` constant and modify `_serialize_breaches()` to apply `[:limit]`. The frontend already paginates (BREACH_PAGE_SIZE=25 in render.js) and cursor pagination (`/api/search/more-breaches`) fetches the rest on demand.

**Lines affected:** ~658-663

### T2 — Sherlock response body bounding (modules/sherlock_wrapper.py)

Add `MAX_BODY_BYTES = 524_288` (512KB) and truncate `resp.text` before text matching in `_check_platform()`. Prevents a single platform returning megabytes of HTML from inflating memory.

**Lines affected:** ~295-298

### T3 — Sherlock async conversion (modules/sherlock_wrapper.py)

Convert `search_username()` from sync (with deprecated `asyncio.new_event_loop()`) to `async def`. CLI fallback (`_try_sherlock_cli`) is sync with `subprocess.run(timeout=120)` — wrap in `asyncio.to_thread()`.

**Lines affected:** ~365-398

### T4 — Remove asyncio.to_thread for Sherlock (api/main.py)

In `_stream_search`, change the Sherlock call from `asyncio.to_thread(search_username, ...)` to direct `await search_username(...)` since it's now async.

**Lines affected:** ~899-900

### T5 — tracemalloc startup + /health/memory endpoint (api/main.py)

Enable `tracemalloc.start(10)` in the `startup()` handler. Add admin-only `GET /health/memory` endpoint exposing RSS, VMS, tracemalloc current/peak, top 15 allocations, and cache stats.

**Lines affected:** startup handler (~442-446), new endpoint after /health

### T6 — Enrich /health with RSS and cache stats (api/main.py)

Add `rss_mb` (process RSS via `psutil.Process()`) and `cache_entries` (TTLCache size) to the existing `/health` response dict. No breaking changes — additive only.

**Lines affected:** ~1530-1559
