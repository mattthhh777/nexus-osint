# Phase 15 Plan 02 — Extract `deps.py`

## Objective

Extrair os 5 símbolos de dependency injection do FastAPI (`security`,
`get_client_ip`, `_check_blacklist`, `get_current_user`, `get_admin_user`)
de `api/main.py` para `api/deps.py`, mantendo backward compat via re-export.

Zero breaking change esperado. Gate: 61/61 testes refactor-scope verdes.

## Read First

- `.planning/phases/15-refactor-main-py-layers/15-CONTEXT.md` — import contract D-05
- `api/main.py:34` — import `HTTPAuthorizationCredentials, HTTPBearer`
- `api/main.py:169-180` — `get_client_ip(request)`
- `api/main.py:285` — `security = HTTPBearer(auto_error=False)`
- `api/main.py:386-466` — bloco `_check_blacklist` → `get_current_user` → `get_admin_user`
- `api/schemas.py` — modelo de leaf module (referência de estilo)
- `api/db.py` — `_db` está exportado? Confirmar import path

## Import Contract (D-05 adaptado)

`api/deps.py` pode importar:
- stdlib: `time`, `typing`
- 3rd party: `aiosqlite`, `fastapi`, `fastapi.security`, `jwt` (PyJWT), `loguru.logger`
- interno: `api.db` (para `_db`) — deps.py está acima de db.py no grafo
- interno: `api.main` → **PROIBIDO** (geraria ciclo)
- interno: `api.schemas` → permitido mas não necessário aqui

`deps.py` **não** é leaf (diferente de schemas.py) — pode ter imports internos de camadas infra (`api.db`).

## Tasks

### Task 1 — Criar `api/deps.py`

**Ação:** Criar arquivo novo `api/deps.py` com os 5 símbolos.

**Conteúdo obrigatório:**

1. Module docstring explicando escopo (providers de Depends() apenas).
2. Imports necessários:
   - `import time`
   - `from typing import Optional`
   - `import aiosqlite`
   - `from fastapi import Depends, HTTPException, Request, status`
   - `from fastapi.security import HTTPAuthorizationCredentials, HTTPBearer`
   - `from loguru import logger`
   - `from api.db import db as _db` (ou caminho correto após conferir db.py)
3. `security = HTTPBearer(auto_error=False)` — instância global
4. `_last_blacklist_warn: list[float] = [0.0]` — estado de rate-limit do warn
5. `get_client_ip(request: Request) -> str` — verbatim de main.py:169-180
6. `async def _check_blacklist(jti: Optional[str]) -> None` — verbatim de main.py:390-427.
   Importar `_decode_token` de main.py? **NÃO** — `_check_blacklist` não chama `_decode_token`,
   apenas recebe `jti` já decodificado. Confirmar no código antes de mover.
7. `async def get_current_user(request, credentials=Depends(security)) -> dict` — verbatim de
   main.py:438-460. **PROBLEMA:** chama `_decode_token` que vive em main.py. Mitigar importando
   `_decode_token` via **pacote de compat temporária**: `from api.main import _decode_token`
   cria **ciclo** (main.py importaria deps.py). Solução: mover `_decode_token` também para
   `deps.py` OU criar `api/security.py` para tokens. **Decisão:** mover `_decode_token` junto
   (mínimo caminho sem criar módulo novo). Adicionar `_decode_token` à lista de símbolos movidos.
8. `async def get_admin_user(user: dict = Depends(get_current_user)) -> dict` — verbatim.

**REVISÃO DE ESCOPO (apurado durante planning):** adicionar `_decode_token` ao Plan 02 porque
`get_current_user` depende dele. Isso também arrasta `_create_token` e `_revoke_token` por
simetria? **NÃO** — `_create_token` é chamado só em `login()` (route layer) e `_revoke_token`
só em `logout()` (route layer). Nenhum dos dois é chamado por dependency. Ficam em main.py até
Plan 04 (routes).
