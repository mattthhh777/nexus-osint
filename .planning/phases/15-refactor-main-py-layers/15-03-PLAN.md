## Decisions locked by this plan

**D-03-01 — `_stream_search` moves verbatim.**
The 510-line SSE generator is transplanted as-is to `search_service.py`. Every helper it calls (`_get_cached`, `_set_cached`, `with_timeout`, `detect_type`, `_serialize_breaches`, `_serialize_stealers`, `_log_search`, `_parse_discord_history`, `_run_spiderfoot`) lives in the same module, so internal references resolve without import changes. External dependencies (`oathnet_client`, `get_orchestrator`, `SpiderFootTarget`, `search_username` left inside the function body, `httpx`, `aiosqlite`, `re`, `json`, `time`, `datetime`, `timezone`, `logger`, `ValidationError`) become module-level imports of `search_service.py`. The `from modules.sherlock_wrapper import search_username` line stays INSIDE the function body — same place it already sits — to avoid surprising import-time cost.

**D-03-02 — `load_dotenv()` inside `api/config.py`.**
`api/config.py` calls `load_dotenv()` at the top, before any `os.environ.get(...)` read. `load_dotenv()` is idempotent (does not overwrite already-set variables), so `api/main.py` keeping its own `load_dotenv()` call is harmless and acts as belt-and-suspenders for any direct `import api.main` entry point. Order of imports does not matter.

**D-03-03 — `_seen_breach_extra_keys` shared via `from ... import` (mutation-safe).**
`api/main.py`'s `breach_extra_keys` route does `sorted(_seen_breach_extra_keys)`; it reads but never reassigns. `_serialize_breaches` (now in search_service) calls `.update()`. Both are mutations of the same `set` object, so `from api.services.search_service import _seen_breach_extra_keys` works — Python shares the reference, not the binding. Explicitly documented here so a future reader does not "fix" it into a getter.

**D-03-04 — `_users_cache` is NOT re-exported to main.py.**
Unlike `_seen_breach_extra_keys`, `_users_cache` is *reassigned* inside `_load_users` (`_users_cache = json.loads(...)`). A `from api.services.auth_service import _users_cache` in main.py would capture the stale initial `None` binding. Main.py accesses the cache only through `_load_users()` / `_save_users()` today — grep confirmed zero direct references in route handlers. Keep it that way.

**D-03-05 — `api/deps.py` consolidates to `api.config`.**
`api/deps.py` currently re-reads `JWT_SECRET` and `JWT_ALGORITHM` from env at import time. With `api/config.py` now being the source of truth, `api/deps.py` switches to `from api.config import JWT_SECRET, JWT_ALGORITHM`. This is an isolated two-line change: value-equivalent, no semantic shift. Ships as its own commit (Task 5) so a regression here is bisectable separately from the larger main.py rewire.

**D-03-06 — Commit granularity: 6 atomic commits, one per task.**
Each task below ships a single commit. Tests stay green at every commit boundary because Tasks 1–4 add new files without wiring them up, Task 5 changes a constant source (value-equivalent), and Task 6 is the moment of truth where main.py switches to the new modules. If Task 6 breaks, revert that single commit — Tasks 1–5 remain merged as scaffolding for a retry.
